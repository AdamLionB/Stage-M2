Singleton 1 - Le score distingue une correcte résolution de singleton d’une mauvaise

type

Singleton, système

description

Le score distingue une correcte résolution de singleton d’une mauvaise
La résolution de singleton peut résulter en trois situations :
  • correcte résolution du singleton
  • incorrecte association du singleton à une chaine
  • incorrecte association du singleton à un autre singleton
L’erreur peut être détecté sur F, P, R
Le score peut :
a- Dans tous les cas distinguer une correcte résolution d’une mauvaise
b- Ne jamais distinguer une correcte résolution d’une mauvaise
c- Aucun des deux

citations

[MUC] does not give any credit for separating out singletons (entities that occur in chains consisting only of one element, the entity itself) from other chains which have been identified.

BAGGA ET BALDWIN, ALGORITHMS FOR SCORING COREFERENCE CHAINS, 1998 (B3)

[MUC] ignores single-mention entities since no link can be found in these entities

LUO, ON COREFERENCE RESOLUTION PERFORMANCE METRICS 2005 (CEAF)

given that [MUC] only takes into account coreference links, the addition of singletons to SYS does not make any difference. It is only when a singleton mention is misclassified in a multi-mention entity that the MUC score decreases.

RECASENS ET HOVY, BLANC: IMPLEMENTING THE RAND INDEX FOR COREFERENCE EVALUATION, 2011

[MUC] has an incorrect preference in ranking coreference outputs. MUC favors the outputs in which entities are over-merged (Luo, 2005). For instance, if we link all the key mentions of the CoNLL 2012 test set into a single response entity, the corresponding MUC scores, i.e. Recall=100, Precision=78.44 and F1=87.91, will be all higher than those of the state-of-the-art system

MOOSAVI ET STRUBE, WHICH COREFERENCE EVALUATION METRIC DO YOU TRUST? A PROPOSAL FOR A LINK-BASED ENTITY AWARE METRIC, 2016 (LEA)

citations

All errors are considered to be equal. The MUC scoring algorithm penalizes the precision numbers equally for all types of errors
Our observation is that some coreference links introduce more error into the resulting equivalence classes that others, and it is important to be able to notice that diference through scoring. Our B-CUBED coreference scoring algorithm overcomes this shortcoming of the MUC-6 algorithm

BAGGA ET BALDWIN, ALGORITHMS FOR SCORING COREFERENCE CHAINS, 1998 (B3)

[MUC] fails to distinguish system outputs with different qualities: the link-based F-measure intrinsically favors systems producing fewer entities, and may result in higher F-measures for worse systems

LUO, ON COREFERENCE RESOLUTION PERFORMANCE METRICS 2005 (CEAF)

[MUC] is indulgent as it is based on the minimal number of missing and wrong links, which often results in counterintuitive results. Classifying one mention into a wrong entity counts as one P and one R error, while completely merging two entities counts as a single R error, although this is further away from the real answer. As a result, the MUC score is too lenient with systems that produce overmerged entities
If all mentions in each document of the MUC test sets2 are linked into one single entity, the MUC metric gives a score higher than any published system (Finkel and Manning 2008).

RECASENS ET HOVY, BLANC: IMPLEMENTING THE RAND INDEX FOR COREFERENCE EVALUATION, 2011

MUC is known to be the least discriminative coreference resolution metric (Bagga and Baldwin, 1998; Luo, 2005; Recasens and Hovy, 2011). The MUC evaluation is only based on the minimum number of missing/extra links in the response compared to the key entities. For instance, MUC does not differentiate whether an extra link merges two singletons or the two most prominent entities of the text. However, the latter error does more damage than the first one.

MOOSAVI ET STRUBE, WHICH COREFERENCE EVALUATION METRIC DO YOU TRUST? A PROPOSAL FOR A LINK-BASED ENTITY AWARE METRIC, 2016 (LEA)

test

On regarde l’évolution du score moyen pour un nombre de perturbation donné ceci suivant le pourcentage de singleton dans le gold
Pour un nombre de mention donné, on génère toutes les partitions que l’on regroupe par nombre de singleton. Pour chaque partition on regarde le score entre elle une variation de celle-ci à n variation près (déplacement)
ex :
pour 1 2 3 4
on génère toutes partitions possible 1 2 3 4, 1 2 34, 1 23 4…. que l’on regroupe par pourcentage p de singleton
pour chacune des partitions on calcul le score moyen de la métrique en apportant n variation (déplacement) la même avec les entités (sans singleton)
